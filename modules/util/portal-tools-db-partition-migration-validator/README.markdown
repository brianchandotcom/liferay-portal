# Database Partition Migration Validator Tool
This tool allows the export and validation of the database information required for a successful partition migration.

## Requirements
- MySQL
- Database user with DDL privileges

## Usage
```
./db_partition_migration_validator.sh <OPERATION_MODE> [OPERATION_PARAMETERS]
```

Operation mode:

- `-e`, `--export` Execute data export.
- `-h`, `--help` Display options.
- `-v`, `--validate` Execute data validation.

Data Export parameters:

- `-d`, `--output-dir <arg>` Output directory. Folder ./exports otherwise.
- `-j`, `--jdbc-url <arg>` JDBC URL.
- `-p`, `--password <arg>` Database user password.
- `-s`, `--schema-name <arg>` Database schema name for the source partition. Default schema name otherwise.
- `-u`, `--user <arg>` Database user name.

Data Validation parameters:

- `-s`, `--source-file <arg>` File with the source partition information generated by the Data Export mode.
- `-t` , `--target-file <arg>` File with the target information generated by the Data Export mode.

### Execution examples

Data Export for the source database partition example:
```
./db_partition_migration_validator.sh -e -j "jdbc:mysql://localhost:3306/sourceDefaultSchema" -u user -p password -s lpartition_1234 -d sourceExports
```

Data Export for the target example:
```
./db_partition_migration_validator.sh -e -j "jdbc:mysql://localhost:3306/targetDefaultSchema" -u user -p password -d targetExports
```

Data Validation example:
```
./db_partition_migration_validator.sh -v -s sourceExports/source.json -t targetExports/target.json
```
