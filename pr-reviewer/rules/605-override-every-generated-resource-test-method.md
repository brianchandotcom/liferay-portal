# 605: Override Every Generated Resource Test Method

In a `*ResourceTest` that extends a `Base*ResourceTestCase` (generated by REST Builder), override every test method the generated base defines and annotate the override with `@Override @Test`. The override delegates to `super.testX()` first to preserve the generated coverage, then calls any private `_testX*` helpers that add scenarios for the same REST path. A new `*ResourceTest` test scenario goes inside the matching override as a `_testX*` helper — never as a new top-level `@Test` method.

**Rationale:** The generated `Base*ResourceTestCase` defines one test method per REST path, so the set of overrides in the subclass tells a reader at a glance which paths the suite exercises and that none was silently skipped. The naming chain — `_testGetSiteFragmentApprovedAndDraft` called from `testGetSiteFragment` — keeps every scenario attached to its path, so adding a new case never disconnects from the path it tests. A new top-level `@Test` method in the subclass severs that link: the reader cannot tell which path it covers, and the corresponding `super.testX()` may end up unreachable.

A violation is a `*ResourceTest` extending a `Base*ResourceTestCase` that does any of:

- adds a top-level `@Test`-annotated method whose name does not match a base test method (the scenario should be a `_testX*` helper called from the corresponding `testX` override);
- omits `@Override` on a method that does match a base test method;
- declares a `_testX*` helper whose `X` prefix does not match the parent override's method name in the same class;
- overrides `testX` without calling `super.testX()` first, dropping the generated coverage.

**Example:** PR `175757` (`AssetEntryResourceTest`) added scenario logic outside the override-plus-`_testX*`-helper structure. The fix is to add an `@Override @Test public void testX() { super.testX(); _testXFoo(); _testXBar(); }` and move the new scenarios into private `_testXFoo` / `_testXBar` helpers whose `X` prefix matches the overridden method.
