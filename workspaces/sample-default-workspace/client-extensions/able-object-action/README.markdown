# Object Action Configuration Example

1. Ensure that DXP has been configured with a **JWK (JSON Web Key) Public and Private Keypair**.

    1. A _sample_ Public and Private Keypair can be generated by visiting [mkjwk.org](https://mkjwk.org/)

    1. On the **RSA** tab use the following values:
        - **Key Size:** 2048
        - **Key Use:** Signature
        - **Algorithm:** RS256
        - **Key ID:** SHA-256
        - **Show X.509:** No

    1. Copy the _Public and Private Keypair_ to a text file

    1. Either:

        1. Configure DXP via System Settings

            1. Click on *Open Applications Menu* (_grid_ icon next to the profile icon.)

            1. Select the _Control Panel_ tab

            1. Select _System Settings_

            1. Under _Security_ select _OAuth2_

            1. Select _Authorization Server Configuration_

            1. Checkmark _Issue JWT Access Token_

            1. Paste the _Public and Private Keypair_ copied to a file earlier

            1. Click _Save_

        1. Configure DXP via configuration file

            1. Create a file called `com.liferay.oauth2.provider.rest.internal.configuration.OAuth2AuthorizationServerConfiguration.config` in the `osgi/configs` directory.

            1. Add the following content to the file:

                ```properties
                oauth2.authorization.server.issue.jwt.access.token=B"true"
                oauth2.authorization.server.jwt.access.token.signing.json.web.key="THE_KEY"
                ```

1. Deploy this client extension:

    `./gradlew :client-extensions:able-object-action:deploy`

1. Build the springboot app in the `extra` directory:

    ```
    cd client-extensions/able-object-action/extra
    ./gradlew build
    ```

1. Login to DXP and retrieve the `Client ID` for the OAuth2Application `able-object-action-oauth-application`

    1. Click on *Open Applications Menu* (_grid_ icon next to the profile icon.)

    1. Select the _Control Panel_ tab

    1. Select _OAuth 2 Administration_

    1. Select _able-object-action-oauth-application_ and copy the _Client ID_ and paste it into a text file

1. Use the Client ID in the following command executed from the `extra`
directory:

    ```
    ./gradlew \
        -Ddxp.domains=localhost \
        -Doauth.client.id=<CLIENT_ID> \
        -Doauth.jwks.uri=http://localhost:8080/o/oauth2/jwks \
        bootRun
    ```

1. Create Object Definition in DXP

    1. Click on *Open Applications Menu* (_grid_ icon next to the profile icon.)

    1. Select the _Control Panel_ tab

    1. Select _Objects_

    1. Click the **+** icon to _Add Object_

    1. Fill the required fields and save

    1. Select the new _Object_ for editing

    1. Under _Scope_ choose a _Panel Category Key_. For now choose _Applications > Content_

    1. Add a required _Field_

    1. Add an _Action_

        1. Specify a _Name_ and switch to _Action Builder_ tab

        1. Select a _Trigger > When_. For now choose _On After Add_

        1. Select an _Action > Then_. From the list select `object-action-executor[function#able-object-action]`

        1. Save

    1. Return to the _Details_ tab and click _Publish_

1. Create an Object Entry

    1. Click on *Open Applications Menu* (_grid_ icon next to the profile icon.)

    1. Select the _Applications_ tab

    1. Select the name of the object created under _Content_

    1. Click the **+** icon to _Add Foo_

    1. Fill required fields and _Save_

    1. The logs of the app started from the `extra` directory should show the
    object activity
